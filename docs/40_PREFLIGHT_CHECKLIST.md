# 導入前チェックリスト（準備物）

最終更新: 2026-02-03

このチェックリストは、導入が途中で詰まるのを避けるためのものです。  
特に **ドメイン・通知先・権限** は後回しにすると運用事故に直結します。

---

## 1. 必須（全員）
- [ ] 研究室の「運用チーム（Platform Ops）」が決まっている（最低2名）
- [ ] 予算責任者（Platform Owner）が決まっている
- [ ] 研究室の運用連絡用メール（共同メアド）がある  
  例: `lab-ops@...`
- [ ] GitHub（Orgまたは個人）の運用方針が決まっている  
  - secretsはGitHubに置かない
  - mainブランチの保護ルール（productionなら推奨）

---

## 2. ドメイン関連（Cloudflareを使う場合）
- [ ] 研究室が管理できるドメインがある（例: `lab.example.org`）
- [ ] ドメインのネームサーバ変更が可能（レジストラの権限がある）
- [ ] サブドメイン命名規則が決まっている（例: `exp-<slug>`）

---

## 3. DigitalOcean（必須）
- [ ] DigitalOcean Team を新規作成する（個人アカウント運用を避ける）
- [ ] Team の **2FA** を必須化する（運用ポリシー）
- [ ] Team Contact Email（運用チームの共同メアド）を設定する（重要）
- [ ] Billing alert の閾値を決める（例: $50 / $80 / $100）

---

## 4. 監視/通知（最低限）
- [ ] 「どこに通知を飛ばすか」が決まっている（メール/Slack）
- [ ] Uptime監視の運用先が決まっている（DigitalOcean Uptimeを使う想定）

---

## 5. データ取り扱い（production前に必須）
- [ ] 収集するデータ項目が決まっている（個人情報の有無）
- [ ] データ保持期間（例: 90日）が決まっている
- [ ] データ閲覧者が定義されている（研究室内に限定する、など）

---

## 6. 次に進む
- DigitalOcean の導入: `60_SETUP_DIGITALOCEAN.md`
- Cloudflare の導入（任意）: `50_SETUP_CLOUDFLARE.md`
- 土台検証（Smoke Test）: `80_SMOKETEST_STEP_BY_STEP.md`
